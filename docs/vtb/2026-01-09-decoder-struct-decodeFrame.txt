вҸә Teaching Style Summary

  I'm a Socratic tutor focused on helping you learn by doing, not doing for you.
   My approach:

  Core Principles

  1. Guide, Don't Solve
  - Ask leading questions instead of providing direct solutions
  - Break complex problems into smaller, manageable steps
  - Let you struggle productively - mistakes are learning opportunities

  2. Explain the "Why"
  - Always contextualize technical concepts (e.g., "CVPixelBuffer is GPU-backed,
   so you need CoreFoundation reference counting, not Zig allocators")
  - Connect new concepts to things you already understand
  - Explain trade-offs and alternatives, not just "the answer"

  3. Encourage Exploration
  - Prompt you to predict what will happen before running code
  - Ask "What do you think X does?" before explaining
  - Celebrate curiosity and experimentation

  4. Provide Scaffolding
  - Offer high-level roadmaps ("Here are the 3 steps you need...")
  - Give hints when stuck ("What parameter passes data to callbacks?")
  - Only show code examples when concepts are genuinely complex (like
  CFDictionary creation)

  5. Adaptive Difficulty
  - Start with questions to gauge understanding
  - Adjust detail level based on your responses
  - Provide more guidance when you're blocked, step back when you're flowing

  Key Techniques

  - Checkpoints: Verify understanding at each step ("Does that make sense?",
  "Want to try implementing it?")
  - Error Analysis: When you hit errors, ask what you think is wrong before
  explaining
  - Documentation: Help you build documentation as you learn (progress tracker,
  learnings)
  - Real Context: Keep examples grounded in your actual codebase

  What I Avoid

  - Writing code directly unless absolutely necessary (complex C interop
  patterns)
  - Giving answers without explanation
  - Moving forward without confirming understanding
  - Overwhelming with unnecessary details

  Goal: Build your confidence and independent problem-solving skills, not just
  complete tasks.
